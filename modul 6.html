06 – Booleans & Operatoren (&& und ||) (6:23)

Logische Verknüpfungen von Bedingungen.

let sonnig = true;
let schirm = false;

console.log(sonnig && schirm); // false (beide müssen true sein)
console.log(sonnig || schirm); // true (eine reicht)
console.log(!sonnig);          // false (Umkehrung)

07 – Booleans & Operatoren (==, ===, <=, ...) (6:04)

Vergleichsoperatoren. Wichtig: == vs. ===

console.log(5 == "5");   // true  (nur Wert)
console.log(5 === "5");  // false (Wert + Typ)
console.log(5 !== "5");  // true
console.log(10 > 5);    // true
console.log(5 >= 5);    // true
console.log(4 <= 3);    // false

③ SEKTION III: DEBUGGER

01 – Was ist der Debugger? (1:49)

Der Debugger in den Browser-DevTools (F12) ermöglicht es, JavaScript Schritt für Schritt auszuführen und Fehler zu finden.

02 – Breakpoints setzen (5:42)

Im DevTools → Sources → Klick auf eine Zeilennummer → Roter Punkt = Breakpoint. Das Skript hält dort an.

function berechne(a, b) {
  let summe = a + b;  // << hier Breakpoint setzen
  return summe;
}
berechne(3, 5);

03 – Die Debugger-Pfeile (7:36)

Step Over (F10): nächste Zeile • Step Into (F11): in Funktion • Step Out: aus Funktion • Resume (F8): weiter bis nächsten Breakpoint

04 – Variablen anzeigen (7:30)

Im Debugger-Panel rechts unter “Scope” sieht man alle aktuellen Variablenwerte live während der Ausführung.

05 – Einführung in Scopes (8:45)

Scopes definieren, wo Variablen sichtbar sind.

let global = "global"; // überall sichtbar

function meineFunc() {
  let lokal = "lokal"; // nur hier sichtbar
  console.log(global); // OK
  console.log(lokal);  // OK
}
meineFunc();
// console.log(lokal); // FEHLER!

06 – Scopes im Debugger sehen (6:15)

Breakpoint innerhalb einer Funktion setzen → Im “Scope”-Panel sieht man “Local” und “Global” getrennt.

07 – Wichtige Tabs des Debuggers (6:40)

Console: Ausgaben & Fehler • Sources: Code & Breakpoints • Elements: HTML live • Network: Netzwerkanfragen

04 – Aufgaben: Arrays (5:28)

// includes
function containsElement(arr, el) { return arr.includes(el); }
console.log(containsElement(["Anna","Ben","Clara"], "Ben")); // true

// indexOf
function findElementIndex(arr, el) { return arr.indexOf(el); }
console.log(findElementIndex([10,20,30,40], 30)); // 2

// shift
function removeFirstElement(arr) { arr.shift(); return arr; }
console.log(removeFirstElement([10,20,30])); // [20,30]

// unshift
function addElementToStart(arr, el) { arr.unshift(el); return arr; }
console.log(addElementToStart([2,3,4], 1)); // [1,2,3,4]

// slice
function getSubArray(arr, s, e) { return arr.slice(s, e); }
console.log(getSubArray([1,2,3,4,5], 1, 4)); // [2,3,4]

// join
function joinArray(arr, sep) { return arr.join(sep); }
console.log(joinArray(["apple","banana","cherry"], ", ")); // apple, banana, cherry

⑤ SEKTION V: WCAG (Barrierefreiheit)

Einleitung (0:30)

Einführung in die WCAG-Sektion: Was ist Barrierefreiheit im Web?

01 – Was ist WCAG? (4:06)

WCAG = Web Content Accessibility Guidelines. Standards für barrierefreie Webseiten, damit Menschen mit Behinderungen das Web nutzen können.

<!-- Gutes Beispiel: Bild mit Alt-Text -->
<img src="foto.jpg" alt="Sonnenuntergang am Meer">

<!-- Schlechtes Beispiel: kein Alt-Text -->
<img src="foto.jpg">

02 – WCAG im Gesetz (4:49)

In Deutschland gilt das Barrierefreiheitsstärkungsgesetz (BFSG). Öffentliche Webseiten und digitale Produkte müssen WCAG-konform sein (Stufe AA).

03 – WCAG Richtlinien Seite (3:17)

Offizielle Seite: wcag.com und w3.org/WAI/WCAG22 – dort sind alle Richtlinien aufgelistet und erklärt.

04 – Seitenchecker (6:39)

Tools zum Prüfen von Barrierefreiheit: WAVE (wave.webaim.org), axe DevTools (Browser-Extension) – zeigen Fehler auf der Seite an.

05 – ARIA Attribute (8:01)

ARIA-Attribute ergänzen HTML um zusätzliche Bedeutung für Screenreader.

<button aria-label="Menü schließen">X</button>

<div role="dialog" aria-modal="true" aria-labelledby="titel">
  <h2 id="titel">Einstellungen</h2>
</div>

<!-- Live-Region für dynamische Inhalte -->
<div aria-live="polite" id="status"></div>

⑦ SEKTION VII: ABSCHLUSSPROJEKT – FOTOGRAM

01 – Projektvorstellung: Fotogram (1:55)

Das Abschlussprojekt ist eine Foto-Galerie-Webseite namens “Fotogram”. Fotos werden dynamisch per JavaScript angezeigt und können nach Kategorien gefiltert werden.

02 – Template-functions (4:07)

Eine Funktion erstellt HTML-Code als String und gibt ihn zurück. So können Bilder dynamisch in die Seite eingefügt werden.

function createImageCard(src, title, category) {
  return `
    <div class="card" data-category="${category}">
      <img src="${src}" alt="${title}">
      <h3>${title}</h3>
    </div>
  `;
}

let gallery = document.getElementById("gallery");
gallery.innerHTML += createImageCard("berg.jpg", "Berge", "natur");

03 – Filterbuttons (6:55)

Buttons filtern die Bilder nach Kategorie, indem sie Klassen hinzufügen/entfernen oder .innerHTML neu befüllen.

function filterBilder(kategorie) {
  let karten = document.getElementsByClassName("card");
  for (let i = 0; i < karten.length; i++) {
    if (kategorie === "alle" || karten[i].dataset.category === kategorie) {
      karten[i].classList.remove("hidden");
    } else {
      karten[i].classList.add("hidden");
    }
  }
}

04 – Urheberrecht Bilder (1:48)

Für eigene Projekte nur lizenzfreie Bilder verwenden. Empfohlene Quellen: unsplash.com, pexels.com, pixabay.com.

05 – File names aus Ordner lesen (1:23)

Dateinamen aus einem Ordner können per JavaScript-Array manuell gepflegt werden. Eine automatische Ordner-Auslesung gibt es im Browser nicht.

let bilder = ["foto1.jpg", "foto2.jpg", "foto3.jpg"];
bilder.forEach(function(bild) {
  gallery.innerHTML += `<img src="bilder/${bild}" alt="">`;
});

06 – Event-Bubbling (6:16)

Events “blubbern” vom inneren Element nach außen. Kann ungewollte Effekte erzeugen.

document.getElementById("elternelement").onclick = function() {
  console.log("Elternelement geklickt");
};

document.getElementById("kindelement").onclick = function(event) {
  event.stopPropagation(); // Bubbling stoppen!
  console.log("Nur Kind geklickt");
};
